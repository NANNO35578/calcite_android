# 260414 Bugfix 汇总

## 一、启动流程

### 核心逻辑
1. **DataStore 读取**：`MainActivity.onCreate` 中通过 `UserRepository.isTokenValid()` 读取 `token` 与 `token_expire_time`。
2. **分支判断**：
   - **Token 不存在或过期**：保持在 `loginFragment`（`nav_graph` 的 `startDestination`）。
   - **Token 有效**：使用 `NavOptions.popUpTo(R.id.nav_graph, true)` 清空回退栈，直接跳转 `homeFragment`，并启动 `SyncWorker` 定时同步。
3. **进入主界面后同步**：`NoteListViewModel.init` 中检测网络状态：
   - **有网**：调用 `refresh()` 拉取 `note/folder/tag/file`，写入 Room 后弹 Toast "已同步"。
   - **无网**：仅观察 Room 本地数据，进入离线模式。

### 关键改动文件
- `MainActivity.kt`：启动时 token 校验与导航。
- `SyncWorker.kt`：每 10 分钟执行同步；token 失效时清空登录状态并发送广播。
- `NoteListViewModel.kt`：`init` 中自动触发首次同步。

## 二、文件树构建逻辑

### 核心规则
- `folder_id == 0 || null` → 根目录。
- **先构建文件夹树，再挂载笔记**：保证文件夹始终排在同级笔记前面。
- **图标区分**：`item_tree_folder.xml` 使用 `ic_folder_24`，`item_tree_note.xml` 使用 `ic_note_24`。

### 实现方式
`NoteListViewModel` 采用 **全量观察 + 递归构建**：
1. **观察全部本地数据**：`combine(folderRepository.getAllFoldersLocal(), noteRepository.getAllNotesLocal(), refreshTrigger)`。
2. **分组映射**：
   - `folderMap = folders.groupBy { it.parent_id ?: 0L }`
   - `noteMap = notes.groupBy { it.folder_id ?: 0L }`
3. **递归构建 `buildForParent(parentId, level)`**：
   - 先遍历该 `parentId` 下的子文件夹（按名称排序），根据 `expandedFolders` 集合决定是否继续递归展开。
   - 再遍历该 `parentId` 下的笔记（按标题排序），生成 `TreeNode.NoteNode`。
4. **展开/折叠**：`toggleFolder()` 只修改 `expandedFolders` 集合并触发 `refreshTrigger`，RecyclerView 通过 `ListAdapter.submitList()` 自动 Diff 刷新。

### 关键改动文件
- `NoteListViewModel.kt`：重写 `buildFullTree` 与 `toggleFolder`。
- `NoteRepository.kt` / `FolderRepository.kt`：新增 `syncAllNotes()`、`syncAllFolders()`，并修复 `folder_id` / `parent_id` 的 `null → 0L` 映射。
- `NoteModels.kt`：将 `Note.folder_id` 与 `NoteDetail.folder_id` 设为 `Long?`，兼容后端返回的 `null`。

## 三、返回逻辑实现

### 统一规则
| 当前页面 | 返回行为 |
|---------|---------|
| `loginFragment` / `registerFragment` | `finish()` 直接退出 App |
| `noteEditorFragment` | `finish()` 直接退出 App |
| 其他页面（`homeFragment`、`searchFragment` 等） | 返回 `noteEditorFragment`，并清空中间回退栈 |

### 实现方式
在 `MainActivity` 中通过 `OnBackPressedDispatcher` 统一拦截：
1. 优先关闭左右抽屉。
2. 根据 `navController.currentDestination.id` 判断当前页面：
   - 登录/注册/编辑器 → `finish()`。
   - 其他 → `navigate(R.id.noteEditorFragment, null, NavOptions.popUpTo(R.id.nav_graph, true))`，确保回退栈中只剩编辑器，再次返回即退出，**禁止多层返回**。

### 关键改动文件
- `MainActivity.kt`：`onBackPressedDispatcher.addCallback` 中的返回分发逻辑。

## 四、附加修复点

### 1. 定时同步与 Token 失效跳转
- `SyncWorker` 在 token 过期时发送广播 `com.calcite.notes.ACTION_TOKEN_EXPIRED`。
- `MainActivity` 注册 `BroadcastReceiver`，收到后清空 DataStore 并跳转登录页。

### 2. 登录后数据加载
- `LoginViewModel.login()` 在登录成功后，依次调用：
  - `noteRepository.syncAllNotes(context)`（调用 `/api/note/list?folder_id=0` 获取全部笔记）
  - `folderRepository.syncAllFolders(context)`（BFS 递归调用 `/api/folder/list` 获取全部文件夹）
- 数据写入 Room 后再触发导航到主界面。

### 3. 左抽屉下拉刷新
- `fragment_note_list.xml` 中把 `RecyclerView` 包裹在 `SwipeRefreshLayout` 内。
- `NoteListFragment` 监听 `viewModel.isRefreshing`，`NoteListViewModel.refresh()` 负责拉取四端数据并停止刷新动画。

### 4. 数据流规范
- 新增 `MainViewModel` 供 `MainActivity` 使用：
  - 底部导航 "新建笔记" 走 `MainViewModel.createNewNote()` → `NoteRepository` → Room。
  - OCR 提交与轮询也统一收敛到 `MainViewModel`。
- **禁止 UI 直接调用网络**；所有网络请求必须经过 ViewModel → Repository → Room 的数据流。
